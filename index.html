📖 CPU Scheduling Study Guide

Core Formulas

  • Turnaround Time (TAT) = Finish Time - Arrival Time
  • Waiting Time (WT) = Turnaround Time - Burst Time
  • Response Time = First Start Time - Arrival Time (Usually same as WT in non-preemptive)

Algorithms Cheat Sheet

FCFS (First Come First Served): Simplest non-preemptive algorithm. Processes run in the exact order they arrive. Prone to the Convoy Effect (short processes stuck behind a long one).
SJF (Shortest Job First): Non-preemptive. Always picks the waiting process with the shortest burst time. Provably optimal for average waiting time, but risks starvation for long processes.
SRTF (Shortest Remaining Time First): Preemptive version of SJF. If a new process arrives with a shorter burst than the currently running process's remaining time, the CPU preempts the current process.
Round Robin (RR): Preemptive. Processes take turns running for a fixed time slice (quantum). If they don't finish, they go to the back of the queue. Great for responsiveness.
Priority Scheduling: Runs highest priority process first. Can be preemptive or non-preemptive. Often requires Aging to solve starvation.
HRRN (Highest Response Ratio Next): Non-preemptive. Ratio = (Waiting Time + Burst) / Burst. Favors shorter jobs but gradually increases priority of long-waiting jobs to prevent starvation.
Multilevel Feedback Queue (MFQ): Complex preemptive algorithm with multiple queues. New processes start at the top priority queue (usually small RR quantum). If they use their full quantum, they are demoted to a lower priority queue with a larger quantum (or FCFS). Punishes CPU hogs and favors interactive I/O bounds.
